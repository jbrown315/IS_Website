How this all works:

Prior to taking any inputs, the Perceptron was fed sample data. V1 Example: Input: [0,1] Expected Output: [1]
Then, each of the inputs will be assigned a weight value between 0 and 1. Ex. [0.5]
After getting the weights, the main code will run, which performs math at the level of multivariate calculus.
Each run through the code is called an epoch. After each epoch, the output will be compared to the expected output.
Once the error calculation has been completed, each weight will be adjusted accordingly.
This is why there are generally 10,000 epochs and why the Perceptron gets much more accurate and precise with more epochs.
Finally, after running the main code X amount of times, the weights have reached their final values.
The Perceptron is then fed an input list, Ex. [1,1]. However, it is not given an expected output.
Using the freshly adjusted weights, the Perceptron will calculate an output that should be close to 0 or 1.
